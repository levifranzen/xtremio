# Contributing to Xtremio

Thank you for considering contributing to Xtremio! 🎉

## How to Contribute

### Reporting Bugs

If you found a bug, please open an issue including:

- Clear description of the problem
- Steps to reproduce
- Expected vs. actual behavior
- Screenshots (if applicable)
- Environment (OS, Python version, etc.)

### Suggesting Improvements

Suggestions are always welcome! Open an issue describing:

- The problem your suggestion solves
- How you envision the solution
- Alternatives you considered

### Pull Requests

1. **Fork** the repository
2. **Clone** your fork:
   ```bash
   git clone https://github.com/Raddons/xtremio.git
   ```
3. **Create a branch** for your feature:
   ```bash
   git checkout -b feature/my-feature
   ```
4. **Make your changes** following best practices
5. **Commit** your changes:
   ```bash
   git commit -m "feat: add new feature X"
   ```
6. **Push** to your branch:
   ```bash
   git push origin feature/my-feature
   ```
7. **Open a Pull Request**

### Code Conventions

- Use PEP 8 for Python code
- Add docstrings for public functions
- Keep functions small and focused
- Add comments when necessary
- Write clear commit messages

### Commit Conventions

We use semantic commit messages:

- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation changes
- `style:` Formatting, semicolons, etc.
- `refactor:` Code refactoring
- `test:` Adding or fixing tests
- `chore:` Maintenance tasks

## Code of Conduct

Be respectful and inclusive. Everyone is welcome!

## Questions?

Feel free to open an issue with your questions!
